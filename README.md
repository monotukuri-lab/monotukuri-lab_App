# ものつくりラボ ポータルアプリ (monotukuri-lab_App)

舞鶴工業高等専門学校「ものつくりラボ」のTA（ティーチングアシスタント）および教員のための業務効率化ポータルスマートフォンアプリです。
Googleのマテリアルデザイン3（Material 3）をベースにした、シンプルかつプレミアム感のあるクリーンなパステルブルーUIを採用しています。

本アプリは、**「TA (学生アシスタント)」と「教員（教師）」の全員が管理者としてのフル権限（全権）を対等に持つシンプルかつフラットな設計**となっています。独立した「システム管理者」などの特別なロールは不要で、全員が責任を持ってラボを運用できます。

---

## 🚀 主な機能と使い方

### 1. ログイン & 自動ログイン維持
- **2つのロール選択**: ログイン画面には「TA (学生)」と「教員」の2つの選択肢のみが配置されています。不要なパスワード入力なしで、すぐに自分の職種とお名前（TAの場合は学籍番号等）でログインできます。
- **セッション永続化**: ログイン情報はスマートフォンの `LocalStorage` に保持され、アプリを閉じて再起動しても自動的にダッシュボード画面が表示されます。

#### 💡 使い方:
1. 起動時のログイン画面で「TA (学生)」または「教員」を選択します。
2. お名前（TAの場合は学籍番号、学科、学年も）を入力して「ログイン」をタップします。
3. 以後はアプリを開くだけで自動的にログインが維持されます。

---

### 2. シフト管理 (カレンダー)
- **勤務時間**: 平日 `16:15 〜 18:15` の時間枠を基準とします。
- **定員制限なしの自由登録**: 1枠あたりの登録人数に制限はなく、全員が自由に参加・脱退を登録できます。
- **特定日の一時削除（全員実行可能）**: 祝日、長期休暇、定期試験期間など、ラボが休館する特定の日付について、**TA・教員の誰もが**その日のシフト枠自体を一時的に削除（休館日設定）できます。

#### 💡 使い方:
1. 底部メニューから「シフト」をタップします。
2. カレンダーから登録・確認したい日付をタップします。
3. **シフトに入る/抜ける**: 「このシフトに入る」または「シフトから抜ける」をタップすると、瞬時に反映されます。
4. **シフト枠の削除（休館日設定）**: 祝日や試験期間などで枠を休止させたい場合、右下の「目のスラッシュマーク（非表示）」ボタンをタップすると、その日付のシフト枠を削除できます。削除した枠は「復活させる」ボタンからいつでも元に戻せます。

---

### 3. 3Dプリンター使用管理
- **利用開始の記録**: 一般利用者が3Dプリンターを使用開始する際、ログイン中のTAや教員のデータ（学籍番号、名前、学年、学科）がフォームに自動的に補完されるため、入力の手間が極限まで減らされています。
- **通常の使用終了**: 使用中のプリンターを選択し、使用者の学籍番号を入力して停止します（既存の仕様）。
- **管理者強制停止 (全員実行可能 / GAS無改造連携)**: 
  - **既存のGAS（`Code.js`）は一切変更せず**に、**TA・教員の誰もが**「学籍番号（パスワード）入力不要」でワンタップ強制終了が可能です。

#### 💡 使い方:
1. 底部メニューから「プリンタ」をタップし、各3Dプリンターの稼働状況（使用中/空き）を確認します。
2. **利用開始**: 空いているプリンターを選択して「利用開始を記録する」をタップ。利用者の学年・学科・学籍番号・氏名を入力して開始します（ログインユーザー自身の情報が自動補完されます）。
3. **利用終了（通常）**: 使用中のプリンターをタップし、利用開始時に入力した学籍番号（Sid）を入力して終了します。
4. **強制停止（全員実行可能）**: 万が一利用者が学籍番号を忘れたり、途中で帰宅してしまった場合など、稼働中のプリンターの右側にある「強制停止」ボタンをタップするだけで、学籍番号を調べることなく瞬時に稼働を終了させることができます。

## 🛠 技術的な仕組み：既存GASに手を加えない「管理者強制停止」

既存の3Dプリンター管理システム（`3D-printer-usage-management/src/Code.js`）は、セキュリティ保持のため終了時に使用者の学籍番号の入力・一致検証が必須となっています。
本アプリでは、この既存の仕組みを変更（破壊）することなく、以下のロジックで安全な「全員管理者による強制停止」を実現しました。

```
[ 強制停止をタップ (TA・教員だれでも可能) ]
                       ↓
[ アプリ側で最新の printerStatus API を取得 ]
                       ↓
[ 稼働中プリンターの status から studentId を自動抽出 ]
                       ↓
[ 抽出した studentId を checkSid パラメータとして endUsage API に送信 ]
                       ↓
[ GAS側は「本人が正しい学籍番号で終了した」と判定し、正常終了処理を実行 ]
```

これにより、既存のGASの検証ロジックをバイパスすることなく、完全に安全かつスマートな強制停止がアプリ側だけで完結しています。

## 💻 開発者向けセットアップ手順

### 必要要件
- Node.js (v18以上推奨)
- npm

### 1. 依存ライブラリのインストール
プロジェクトのルートディレクトリで以下のコマンドを実行します。
```bash
npm install
```

### 2. ローカル開発サーバーの起動
ローカルでアプリを起動し、リアルタイムでの開発やデバッグを行います。
```bash
npm run dev
```
起動後、ブラウザで表示されるローカルURL（例: `http://localhost:5173`）にアクセスします。PCで動作させる場合も、モバイル実機サイズに最適化されたフレーム付きで美しく表示されます。

### 3. プロダクション用ビルド
本番公開用の最適化された静的ファイルを生成します。
```bash
npm run build
```
ビルドに成功すると、プロジェクト直下の `dist/` ディレクトリに HTML, CSS, JavaScript ファイルが出力されます。

## 🌐 本番GAS（Google Apps Script）との連携方法（本実装手順）

本アプリは、既存の3Dプリンター管理スプレッドシート（原本）をそのまま活用し、さらにシフト管理とお知らせ管理を自動統合する**「統合API GASスクリプト (`Api.gs`)」**を用いて本番稼働させます。

### 1. スプレッドシート側でのAPI設置手順

1. 既存の3Dプリンター管理スプレッドシート（`https://docs.google.com/spreadsheets/d/1TJwyivkS81Yf6gR7vZbmxnH5D3y1mrf-kZq_n6tar5o/edit`）を開きます。
2. メニューの **「拡張機能」 > 「Apps Script」** を開きます。
3. 左側のファイルリストの「＋」ボタンをクリックして「スクリプト」を追加し、名前を **`Api`** とします（自動的に `Api.gs` が作成されます）。
4. 本プロジェクトの [gas/Api.gs](file:///c:/Users/106no/Documents/GitHub/monotukuri-lab_App/gas/Api.gs) の中身をすべてコピーし、作成した `Api.gs` エディタ内に丸ごと貼り付けて保存（`Ctrl + S`）します。
5. エディタ右上の **「デプロイ」 > 「新しいデプロイ」** をクリックします。
6. 「種類の選択」の歯車アイコンから **「ウェブアプリ」** を選択し、以下のように設定します：
   - **説明**: `ものつくりラボ アプリ統合API`
   - **次のユーザーとして実行**: `自分 (あなたの高専Googleアカウント)`
   - **アクセスできるユーザー**: `全員` (※アプリと非同期通信するために必須の設定です)
7. 「デプロイ」ボタンを押し、発行された **「ウェブアプリのURL」**（`https://script.google.com/macros/s/.../exec`）をコピーします。

> [!TIP]
> **データベース（シート）の自動初期化機能について**
> 
> コピーした `Api.gs` は非常に親切な自動初期化機能を備えています。
> スプレッドシート内に「シフト」や「お知らせ」という名前のシートが存在しない場合、アプリから最初のアクセス（または設定画面での接続テスト時）に**自動的にシートを新規作成し、必要な初期ヘッダーとお知らせのサンプルデータを流し込みます。** 
> ユーザー様側で手動でシートを作成したりカラムを準備したりする必要は一切ありません！

---

### 2. スマホアプリ側での接続手順

1. 本アプリの画面底部メニューから **「設定（マイ設定）」** をタップします。
2. 「GAS API連携設定」の入力フォームに、上記の手順で取得した **「ウェブアプリのURL」** を貼り付けます。
3. **「設定を保存」** をタップします。
4. 保存後に表示される **「接続テストを実行」** ボタンをタップします。
5. 画面に 「🎉 接続テスト成功！本番環境と正常に同期しています」 と表示されれば、本実装は完了です！

以後は、3Dプリンター的稼働状況、シフトカレンダーの登録・削除、お知らせの取得など、すべてのデータがスプレッドシートとCORSエラーなく安全にリアルタイム同期されるようになります。

> **⚠️ 注意（フォールバック動作）:** GAS API的連携URLが未設定、または通信エラーが発生した場合、アプリは自動的にLocalStorageベースの「ローカルモックモード」で安全に動作し続けます。これにより、オフライン環境やテスト環境でもアプリが完全に機能停止することはありません。
